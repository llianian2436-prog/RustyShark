# RustyShark 项目说明文档

`RustyShark` 是一款基于 Rust 语言开发的终端网络数据包捕获与分析工具。项目通过终端文本用户界面（TUI）展示网络数据包列表、协议解析结果和操作按钮，支持网卡选择、BPF 过滤、数据包浏览、协议详情查看以及 PCAP 文件导出等功能。

本项目主要用于 Rust 课程实践，重点体现 Rust 在系统编程、并发处理、错误管理和终端交互开发中的应用。

---

## 一、依赖说明

项目的编译与运行依赖主要包括系统环境依赖和 Rust 项目依赖两部分。

### 1.1 系统环境依赖

由于本项目需要捕获本机网卡数据包，因此运行前需要满足以下条件：

1. **操作系统**

   本项目主要在 Windows 10 / Windows 11 x86_64 环境下开发和测试。

2. **Npcap 驱动**

   Windows 平台下需要提前安装 Npcap 驱动程序。安装时建议勾选：

   ```text
   Install Npcap in WinPcap API-compatible Mode
   ```

   如果未安装 Npcap，程序可能无法正常扫描或打开本地网卡。

3. **Npcap SDK 链接库**

   在 Windows 平台编译时，项目可能需要链接 Npcap SDK 中的相关库文件，例如：

   ```text
   Packet.lib
   wpcap.lib
   ```

   如果编译时出现链接错误，需要将 Npcap SDK 的 `Lib` 路径配置到系统环境变量 `LIB` 中。

4. **管理员权限**

   抓包程序通常需要较高的系统权限。建议使用管理员身份打开 PowerShell 或 CMD 后，再执行编译和运行命令。

### 1.2 Rust 项目依赖

项目依赖由 `Cargo.toml` 统一管理，主要第三方库如下：

| 依赖库         |       版本 | 作用                                |
| ----------- | -------: | --------------------------------- |
| `pcap`      |  `1.3.0` | 调用底层抓包接口，负责网卡打开、数据包捕获和 BPF 过滤规则设置 |
| `tokio`     | `1.52.3` | 提供异步运行时和多线程任务调度能力                 |
| `ratatui`   | `0.26.3` | 用于构建终端文本用户界面                      |
| `crossterm` | `0.27.0` | 用于终端控制、键盘事件和鼠标事件捕获                |
| `thiserror` | `1.0.69` | 用于定义和管理项目中的错误类型                   |

具体版本以项目中的 `Cargo.toml` 文件为准。

## 二、编译运行方式

在运行项目前，请先确认已经安装 Rust 工具链，并能够正常使用 `cargo` 命令。

建议在项目根目录下，以管理员身份打开 PowerShell 或 CMD，依次执行以下命令。

### 2.1 配置 Npcap SDK 链接路径

如果本机尚未将 Npcap SDK 的库目录加入系统环境变量，编译时可能会出现找不到 `Packet.lib` 或 `wpcap.lib` 的链接错误。

此时可以在 PowerShell 中临时配置 `LIB` 环境变量：

```powershell
$env:LIB="C:\NpcapSDK\Lib\x64"
```

其中 C:\NpcapSDK\Lib\x64 需要替换为本机实际的 Npcap SDK 路径。

例如，如果 Npcap SDK 解压在：

D:\Tools\Npcap-SDK

则可以执行：

```powershell
$env:LIB="D:\Tools\Npcap-SDK\Lib\x64"
```

该配置只对当前 PowerShell 窗口有效。关闭终端后需要重新设置；如果希望长期生效，可以将该路径添加到 Windows 系统环境变量 LIB 中。
### 2.2 编译并运行项目

执行以下命令编译并启动程序：

```powershell
cargo run
```

如果编译和运行过程中出现与 Npcap 相关的错误，请优先检查 Npcap 是否已正确安装，以及 Npcap SDK 的链接库路径是否配置正确。

## 三、使用方法

程序启动后，会依次进入初始化配置阶段和终端界面交互阶段。

### 3.1 初始化配置阶段

#### 3.1.1 选择网卡设备

程序启动后会自动扫描当前主机中的可用网卡，并在控制台中输出网卡编号、名称和描述信息。

用户根据提示输入对应的网卡编号即可选择抓包网卡。如果直接按回车键，则程序会使用默认网卡编号。默认编号以程序实际提示为准。

#### 3.1.2 输入 BPF 过滤表达式

选择网卡后，程序会提示用户输入 BPF 过滤表达式。常见示例如下：

```text
tcp
udp
ip
ip6
ip and tcp
ip6 and tcp
```

如果不需要过滤，可以直接按回车键，程序将进入全流量捕获模式。

### 3.2 主界面交互阶段

完成初始化后，程序会进入终端全屏界面。主界面主要由数据包列表、协议详情区域和底部控制按钮区域组成。

#### 3.2.1 数据包列表操作

数据包列表用于展示当前捕获到的数据包基本信息，例如时间、协议类型、源地址、目的地址、端口号和长度等。

支持以下操作：

1. **键盘上下移动**

   使用方向键 `Up` / `Down` 可以逐行移动当前选中的数据包。

2. **鼠标滚轮浏览**

   在数据包列表区域滚动鼠标滚轮，可以向上或向下查看历史数据包。

3. **鼠标点击选中**

   使用鼠标左键点击列表中的某一行，可以直接选中对应数据包。

### 3.2.2 滚动条操作

程序在数据包列表右侧提供滚动条，用于快速定位数据包列表中的不同位置。

支持以下操作：

1. **单行微调**

   点击滚动条顶部或底部的箭头区域，可以将当前选中位置向上或向下移动一行。

2. **拖动定位**

   按住滚动条滑块并上下拖动，可以按照列表总长度进行比例定位，快速跳转到数据包列表中的指定区域。

### 3.2.3 协议详情查看

当用户选中某个数据包后，下方协议详情区域会展示该数据包的解析结果。

目前主要支持以下信息展示：

1. **数据链路层信息**

   包括以太网帧类型、源 MAC 地址、目的 MAC 地址等。

2. **网络层信息**

   包括 IPv4、IPv6、ARP 等协议的基本字段，例如源 IP 地址、目的 IP 地址和协议类型。

3. **传输层信息**

   包括 TCP、UDP 等协议的端口号、控制位、序列号、数据长度等信息。

协议详情区域会根据当前选中的数据包自动更新，用户不需要额外输入命令。

### 3.2.4 底部控制按钮操作

底部区域提供若干控制按钮，用户可以通过鼠标左键点击触发对应操作。

| 按钮    | 功能说明                                   |
| ----- | -------------------------------------- |
| 开始/恢复 | 启动或恢复后台抓包任务，使数据包继续追加到列表中               |
| 暂停抓包  | 暂停界面中的数据包更新，便于用户查看已有数据                 |
| 停止抓包  | 停止当前抓包会话                               |
| 一键清空  | 在非运行状态下清空当前缓冲区中的数据包                    |
| 导出文件  | 将当前缓冲区中的原始数据包导出为 `rusty_shark.pcap` 文件 |
| 退出程序  | 退出终端界面并恢复控制台状态                         |

导出的 `rusty_shark.pcap` 文件会保存在项目根目录下，可使用 Wireshark 等工具打开进行进一步分析。

除点击底部退出按钮外，用户也可以在程序运行过程中按下 `q` 或 `Q` 键退出程序。

## 四、注意事项

为了保证程序能够稳定运行，使用和维护项目时需要注意以下问题。

### 4.1 BPF 过滤表达式必须合法

用户输入的过滤表达式需要符合 BPF 语法规则。如果表达式拼写错误或格式不正确，底层抓包接口可能会返回错误，导致抓包任务无法启动。

例如，在过滤 IPv6 流量时，应使用：

```text
ip6
```

而不是：

```text
ipv6
```

如果需要捕获 IPv6 TCP 流量，可以输入：

```text
ip6 and tcp
```

如果不确定过滤规则是否正确，可以先直接按回车键进入全流量捕获模式，确认程序能够正常启动后再添加过滤条件。

### 4.2 正常退出程序

本项目运行时会进入终端原始模式，并接管部分键盘、鼠标和界面显示控制。如果程序没有正常退出，可能会导致当前控制台出现字符无法正常显示、回车异常或界面状态未恢复等问题。

因此，建议使用以下方式正常退出程序：

1. 点击底部的“退出程序”按钮；
2. 按下键盘上的 `q` 或 `Q` 键。

不建议在程序运行时直接点击控制台窗口右上角关闭按钮，也不建议频繁使用 `Ctrl + C` 强制中断程序。

如果出现控制台显示异常，可以关闭当前终端窗口后重新打开，或重新启动开发环境。

### 4.3 注意缓冲区内存占用

程序会将捕获到的数据包保存在内存缓冲区中。若在高流量网络环境下长时间运行，内存占用可能会逐渐增加。

如果需要长时间抓包，建议定期导出并清空数据。清空缓冲区前，建议先执行以下操作之一：

1. 点击“暂停抓包”；
2. 点击“停止抓包”。

在抓包仍处于运行状态时，不建议直接清空缓冲区，以避免数据更新和界面读取同时发生，影响程序稳定性。

### 4.4 管理员权限问题

如果程序无法打开网卡，或者启动后无法捕获数据包，请检查当前终端是否以管理员身份运行。

在 Windows 平台下，建议右键点击 PowerShell 或 CMD，选择“以管理员身份运行”，然后进入项目目录执行：

```powershell
cargo run
```

### 4.5 PCAP 文件导出说明

点击“导出文件”按钮后，程序会将当前缓冲区中的数据包保存为：

```text
rusty_shark.pcap
```

该文件位于项目根目录下。导出文件可使用 Wireshark 打开。若导出后文件内容为空，请检查是否已经成功捕获到数据包，以及当前缓冲区中是否存在数据。
